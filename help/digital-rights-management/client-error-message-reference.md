# Referência de mensagem de erro de cliente DRM {#client-error-message-reference}

Os erros do cliente DRM são um subconjunto dos erros do lado do cliente TVSDK, com os códigos de erro relacionados ao DRM variando de 3300 a 3399.

## Erros de cliente DRM {#drm-client-errors}

| Código de erro | Mnemônico | Solução |
|---|---|---|
| 3300 | Comprovante inválido | O que o software do distribuidor deve fazer: <ul><li> Se você estiver usando o Google Chrome, estiver no modo Incógnito e a versão do Flash Player for anterior à 11.6, este erro pode ocorrer. Recomendamos que o reprodutor verifique o número da versão do navegador e aconselhe o usuário a sair do modo Incógnito.</li><li> Solicite a licença novamente. Se a solicitação for bem-sucedida, não será necessário registrar nem escalar. Se a solicitação não tiver êxito, registre o conteúdo que causou o erro. `subErrorId` contém um erro de linha, se presente.</li></ul> O que o distribuidor deve fazer: <ul><li> Se as tentativas forem malsucedidas em configurações diferentes do Chrome com Flash inferior à versão 11.6, uma falha pode ter ocorrido no pacote.</li><li>Verifique se o problema é específico de determinado conteúdo e reempacote.</li></ul> |
| 3301 | Falha na autenticação | O servidor não pôde autenticar ou autorizar o cliente.<ul><li>O software do distribuidor deve tomar as medidas necessárias para restabelecer as credenciais do usuário ou orientá-lo a adquirir acesso ao conteúdo.</li><li>O distribuidor deve confirmar que o seu mecanismo de autorização e autenticação está a funcionar corretamente. Se os distribuidores não estiverem planejando usar os recursos de autenticação ou autorização, eles deverão verificar se a política do conteúdo ofensivo requer autenticação e consultar Diagnóstico de discrepâncias de política/licença.</li></ul> Para obter mais informações sobre esse código de erro, consulte [Causas e resolução do erro DRM 3301](https://forums.adobe.com/thread/1277149). |
| 3302 | RequireSSL | Para o Primetime DRM 4.0 e superior, esse erro é lançado no iOS quando o URL da chave remota não usa HTTPS como esquema. HTTPS é necessário. <ul><li>Se o distribuidor estiver usando uma versão anterior à versão 4 do Primetime DRM ou, pelo menos, à versão 4, mas a plataforma não for iOS, o software do distribuidor deverá registrar o erro. Esse erro é lançado somente no iOS.</li><li>Se o software do distribuidor for pelo menos o Primetime DRM versão 4 e a plataforma for o iOS, os distribuidores deverão alterar o URL do servidor de chave remota que estão usando para HTTPS. Se eles estiverem usando apenas HTTP, talvez os distribuidores precisem configurar um servidor HTTPS. Caso contrário, os distribuidores precisarão enviar as informações registradas para o Adobe e encaminhar o problema.</li></ul> |
| 3303 | ContentExpired | O conteúdo que você está visualizando expirou de acordo com as regras definidas pelo provedor de conteúdo.  `subErrorId` contém um erro específico do cliente ou erro de linha. <ul><li>O software do distribuidor deve tentar readquirir uma licença do servidor uma vez para determinar se uma nova licença não expirada está disponível. Se nenhuma licença estiver disponível ou a licença tiver expirado, permita que o usuário adquira uma nova licença ou informe ao usuário que o conteúdo não pode ser assistido.</li></ul> Se o conteúdo tiver sido empacotado com uma política com uma data de expiração/término expirada e os registros do servidor de licença relatarem uma PolicyEvaluationException, indique que a Data de término da política expirou (Código de erro do servidor 303). Verifique os arquivos de log do servidor.<br>Se possível, os clientes devem verificar a política usada durante o empacotamento para ver se ela expirou. A ferramenta de linha de comando Java é: <ul><li>java -jar libs/AdobePolicyManager.jar</li><li> demonstração detalhada.pol</li><li>O distribuidor deve confirmar se as datas de expiração da licença estão configuradas conforme esperado.</li></ul> Para obter mais informações sobre esse código de erro, consulte [3303 (Conteúdo expirado) com AMS/FMS usando um Live Stream?](https://forums.adobe.com/thread/1300813) |
| 3304 | AuthorizationFailed | O usuário atual não está autorizado a visualizar conteúdo. Faça logon como um usuário diferente.<br>Para obter mais informações sobre esse código de erro, consulte Código de erro 3301. |
| 3305 | ServerConnectionFailed | A conexão com os servidores de licença ou de domínio atingiu o tempo limite devido a um atraso de rede ou porque o cliente estava offline. Geralmente, subErrorId contém o código de retorno HTTP.<ul><li>O software do distribuidor deve tentar uma conexão de rede com um servidor em boas condições. Se a tentativa falhar, peça para o usuário se reconectar à rede. Se a tentativa for bem-sucedida, registre-a.</li><li>O distribuidor deve verificar se as licenças e os servidores de domínio em uso estão online e visíveis na rede do cliente.</li></ul>Para obter mais informações sobre esse código de erro, consulte [DRM 3305 ServerConnectionFailed causa e resolução](https://forums.adobe.com/thread/1284947.) |
| 3306 | ClientUpdateRequired | O cliente atual não pode concluir a ação solicitada, mas um cliente atualizado pode conseguir concluir a solicitação.<br>Isso pode ter várias causas:<ul><li>Um domínio compartilhado que não está disponível neste cliente foi usado. Esse provavelmente é o caso quando a reprodução funciona no Chrome, mas não em qualquer outro navegador e vice-versa.<br>**Nota:** O Chrome usa uma chave PHDS/PHLS diferente da usada pelos outros navegadores.</li><li>O aplicativo está tentando adicionar várias DRMSessions ao ser executado em uma versão do iOS anterior à 5.0.</li><li>Os metadados têm uma versão de 3 ou superior quando há suporte apenas para a versão 2.</li><li>O software do distribuidor deve alertar o usuário e interromper a operação. Se o software tiver uma maneira de determinar se um upgrade está disponível, direcione o usuário para esse upgrade da maneira apropriada para a plataforma.</li><li>Se o problema ocorrer devido a um domínio compartilhado, o distribuidor precisará verificar com o Adobe se há um tempo de execução ou biblioteca atualizada. No caso do tempo de execução do Flash, o distribuidor pode forçar a atualização em seu aplicativo diretamente. No caso de uma biblioteca, o distribuidor precisará obter uma biblioteca atualizada, reconstruir o aplicativo e implantá-la nos usuários.</li></ul>Se o problema ocorrer devido a várias DRMSessions, o distribuidor precisará atualizar o aplicativo para verificar o número da versão do iOS antes de adicionar várias DRMSessions. Ou eles podem restringir a distribuição de seus aplicativos ao iOS versão 5 e superior.<br>Se o problema ocorrer porque a versão dos metadados é superior à versão 2, o problema provavelmente são metadados corrompidos. Eles podem tentar reconstruir os metadados e observar os resultados. Se o problema persistir, registre o problema e encaminhe para Adobe.<br>Para obter mais informações sobre esse código de erro, consulte [Como corrigir um código de erro 3306 DRMErrorEvent](https://forums.adobe.com/thread/1266675). |
| 3307 | InternalFailure | Isso geralmente representa um erro no código DRM do Primetime e é inesperado, a menos que haja um erro conhecido, pois below.subErrorId contém um erro específico do cliente ou erro de linha.<ul><li>Se o navegador for o Chrome no Windows e a versão do Flash for 11.6 (SWF versão 19 ou superior), o software do distribuidor deve supor que o usuário pressionou Negar na barra de informações e tratar o mesmo que um 3368.</li><li>Se o 3307 ocorrer quando o navegador não for o Chrome ou a versão do Flash não for a 11.6, o distribuidor deverá encaminhar para o Adobe.</li></ul>**Nota:** 3307:1107296344 (FailedToGetBrokerHandle) pode acontecer com as versões 24-28 do navegador Chrome. |
| 3308 | WrongLicenseKey | Esse erro é lançado sempre que a licença que está sendo usada contém a chave errada para descriptografar o conteúdo. O subErrorIdcontém um erro específico do cliente ou erro de linha.<br>Parece haver apenas duas maneiras de gerar esse erro:<ul><li>O cliente modificou a ferramenta Adobe padrão para gerar licenças (por exemplo, a estrutura Java do servidor de licenças). Nesse caso, a licença contém uma chave incorreta que pode não corresponder a nenhum conteúdo.</li><li>O cliente emitiu várias licenças com a mesma ID de licença. Nesse caso, há várias licenças disponíveis no cliente que correspondem aos metadados do conteúdo, e o código DRM do Primetime selecionou a incorreta para uso.</li><li>O software do distribuidor deve tentar readquirir a licença do servidor.</li><li>Se nenhuma licença estiver disponível ou a licença tiver expirado, forneça um fluxo de trabalho para o usuário adquirir uma nova licença ou informe ao usuário que o conteúdo não pode ser assistido e registre o problema.</li><li> Se era um conteúdo vinculado a um domínio (para AIR), forneça uma maneira de o usuário ingressar no domínio.</li><li>O distribuidor deve:</li><li> Verifique se eles não personalizaram as partes de emissão de licença do servidor de licenças do Primetime DRM.</li><li>Verifique se estão emitindo IDs de licença exclusivas para todas as licenças.</li><li>Escalone o problema com o Adobe.</li></ul> |
| 3309 | CorruptedAdditionalHeader | Isso ocorrerá se o cabeçalho tiver mais de 65536 bytes.<ul><li>O software do distribuidor deve registrar qual parte do conteúdo causou o erro.</li><li>O distribuidor deve confirmar que o erro é reproduzível com partes específicas do conteúdo. Reempacotar conteúdo corrompido.</li></ul> |
| 3310 | AppIDMismatch | O aplicativo não está na lista de permissões. Android, iOS ou Flash SWF.<br>subErrorId: 1000942; Erro ao reproduzir fluxo protegido. Erro de FAXS.<br>Também é possível que o cliente esteja relatando uma sequência vazia para pubID (ID do editor do aplicativo).<br>**Android:** O aplicativo Android não corresponde ao aplicativo em uso. Lembre-se de que o diretório do keystore de depuração no Android geralmente é diferente do diretório do keystore de versão.<br>**iOS:** Consulte a [Lista de permissões seu aplicativo iOS](/help/programming/tvsdk-3x-ios-prog/ios-3x-drm-content-security/ios-3x-allowlist-your-ios-application.md) documentação no guia do TVSDK iOS. |
| 3312 | LicenseIntegrity | Baixe a licença do servidor novamente. |
| 3313 | WriteMicrosafeFailed | Esse problema ocorre quando o sistema não consegue gravar no sistema de arquivos. subErrorIdcontém um erro específico do cliente ou erro de linha.<br>No Microsoft Windows, o erro 3313 pode ser lançado pelo flash player do plug Ative X ou NPAPI quando o conteúdo criptografado tem uma licenseID ou uma policyID muito longa. Isso se deve ao comprimento máximo do caminho no Windows. (O plug-in Pepper não tem esse problema.)<ul><li>O software do distribuidor deve solicitar que o usuário confirme se seu diretório de usuário não está bloqueado nem em um volume que esteja cheio ou bloqueado.</li><li>Se o distribuidor estiver usando o AIR, em vez do Flash, o problema pode ser causado por uma limitação de comprimento de caminho. Os distribuidores devem encurtar o nome do aplicativo AIR para algo razoável.</li></ul> |
| 3314 | MetadadosDRM Corrompidos | Esse erro geralmente indica que o conteúdo foi empacotado com certificados PKI de teste e o reprodutor é criado com PKI de produção, ou vice-versa.<br>O subErrorId contém um erro específico do cliente ou erro de linha.<ul><li>O software do distribuidor deve registrar qual parte do conteúdo causou o erro.</li><li>O distribuidor deve confirmar que o erro é reproduzível com conteúdos específicos.</li></ul>Talvez seja necessário reempacotar o conteúdo com falha. |
| 3315 | Permissão negada | Há bugs conhecidos nos quais esse código de erro pode ser lançado quando um 3305 é pretendido. Para obter mais informações, consulte o Código de erro 3305.<br>Um SWF remoto carregado pelo AIR não tem permissão para acessar a funcionalidade DRM do Primetime. Esse código de erro também pode ser lançado se ocorrer um erro de segurança durante o acesso à rede. Os exemplos incluem o servidor de destino no qual o cliente não pode se conectar usando crossdomain.xml ou o crossdomain.xml não está acessível.<br>Para obter mais informações, consulte [Possível causa raiz e resolução do erro DRM 3315](https://forums.adobe.com/thread/1266592). |
| 3317 | AXS_LoadAdobeCPFailed | **Importante:** Esse é um erro raro e geralmente não ocorre em um ambiente de produção.<br>Se o erro ocorrer, siga um destes procedimentos:<ul><li>Se você estiver usando o AIR, reinstale o AIR.</li><li>Se estiver usando o Flash Player, baixe os módulos do AdobeCP novamente.</li></ul> |
| 3318 | AdobeCPVersion incompatível | **Importante:** Esse é um erro raro e geralmente não ocorre em um ambiente de produção.<br>Se o erro ocorrer, siga um destes procedimentos:<ul><li>Se você estiver usando o AIR, reinstale o AIR.</li><li>Se estiver usando o Flash Player, baixe os módulos do AdobeCP novamente.</li></ul> |
| 3319 | AdobeCPGetAPI ausente | **Importante:** Esse é um erro raro e geralmente não ocorre em um ambiente de produção.<br>Se o erro ocorrer, siga um destes procedimentos:<ul><li>Se você estiver usando o AIR, reinstale o AIR.</li><li>Se estiver usando o Flash Player, baixe os módulos do AdobeCP novamente.</li></ul> |
| 3320 | HostFailed | **Importante:** Esse é um erro raro e geralmente não ocorre em um ambiente de produção.<br>Se o erro ocorrer, siga um destes procedimentos:<ul><li>Se você estiver usando o AIR, reinstale o AIR.</li><li>Baixe o AdobeCP e o Flash Player novamente porque qualquer solução pode estar fora de sincronia.</li></ul>O aplicativo só precisa atualizar o Flash Player, o que resulta no download do AdobeCP novamente. |
| 3321 | I15nFailed | Falha no processo de provisionamento do cliente com chaves. O subErrorId contém um erro de linha, específico do servidor ou do cliente.<ul><li>O software do distribuidor deve repetir a operação pelo menos uma vez.</li></ul>Se você estiver usando o Google Chrome no Windows, forneça uma explicação sobre como permitir o acesso ao plug-in que não está em uma sandbox. Para obter mais informações, consulte Acesso negado à unsandbox do Google Chrome.<ul><li>O distribuidor deve concluir uma das seguintes tarefas:</li><li>Se o erro for consistente em todas as plataformas, você deve desescalonar o problema com o Adobe.</li><li>Se o erro estiver limitado ao Chrome no Windows, oriente o usuário a permitir acesso a plug-in fora da área restrita.</li></ul>Os distribuidores devem atualizar seu SWF para a versão 19 ou posterior. Para o erro 3321 específico do Chrome, um erro 3368 é lançado. O erro 3368 pode ser tratado mais especificamente pelo software do distribuidor. Essa alteração foi introduzida no canal Chrome Stable versão 26.0.1410.43.<br>**Nota:** O erro 3321:1090519056 pode ocorrer com o Flash Playerversions 11.1 a 11.6. Recomendamos que você atualize para a versão mais recente do Flash Player.<br>Para obter mais informações, consulte [Erro DRM 3321 Causes &amp; Resolution](https://forums.adobe.com/thread/1277138). |
| 3322 | FalhaDeAssociaçãoDeDispositivo | O dispositivo não parece corresponder à configuração que estava presente quando inicializado. subErrorId contém um erro de linha ou específico do cliente.<br>O software do distribuidor deve executar uma das seguintes tarefas:<ul><li>Se o dispositivo não estiver usando o Flash Player e estiver usando o AIR, o iOS e assim por diante, `callDRMManager.resetDRMVouchers()`.</li></ul>Se o problema ocorrer no iOS em uma fase de desenvolvimento, peça ao desenvolvedor para confirmar se o problema é observado ao alternar entre builds que foram baixadas de sistemas de distribuição de pré-lançamento de terceiros (por exemplo, HockeyApp) e uma build local do Xcode. Os atributos de uma instalação anterior não são totalmente substituídos ao alternar entre uma build distribuída do HockeyApp e uma build do Xcode. Essa situação pode acionar o erro 3322.<br>Para resolver esse problema, o desenvolvedor deve remover a build mais antiga do dispositivo antes de instalar a nova build.<ul><li>Se o dispositivo estiver usando o Flash Player e não puder ser usado com um código de erro 3322 ou 3346, consulte as instruções do Adobe sobre como redefinir programaticamente o armazenamento de licenças DRM em [Erro de DRM 3322/3346/3368 no Chrome (problemas na barra de informações)](https://forums.adobe.com/message/5535907#5535907).</li></ul>Este erro não é esperado com frequência. Em ambientes corporativos que usam perfis móveis, se um usuário estava visualizando conteúdo protegido por DRM, a probabilidade de ocorrência do erro 3322 aumenta à medida que o usuário faz logon em máquinas diferentes. Se possível, o distribuidor deve tentar obter essas informações do usuário.<br>Se o erro ocorrer com frequência, encaminhe para Adobe. Você deve notificar o Adobe se a redefinição do armazenamento de licenças resolveu (ou não) o problema e informar ao Adobe em quais navegadores o erro está ocorrendo.<br>Para obter mais informações, consulte os seguintes artigos:<ul><li>https://forums.adobe.com/message/5520902</li><li>https://forums.adobe.com/message/5535911</li><li>https://forums.adobe.com/message/5748618</li><li>https://forums.adobe.com/message/6061165</li></ul> |
| 3323 | CorruptGlobalStateStore | Os arquivos usados pelo cliente DRM foram modificados inesperadamente. O subErrorIdcontém um erro de linha ou específico do cliente.<ul><li>O software do distribuidor deve orientar o usuário a redefinir da mesma forma que para o Código de erro 3322.</li><li>Se o GlobalStore estiver falhando a uma taxa maior que a taxa de falha esperada dos discos rígidos da sua base de usuários, transfira o problema para o Adobe.</li></ul> |
| 3324 | MachineTokenInvalid | Talvez o servidor de licenças não possa se conectar ao servidor CRL (Lista de Certificados Revogados) para atualizar seus arquivos CRL, ou a máquina cliente está solicitando uma licença/autenticação que foi revogada pelo servidor de licenças.<br>Nos registros do servidor, um código de erro 111 é MachineTokenInvalid. No entanto, no nível do cliente, o código de erro 111 é convertido no código de erro 3324.<br>O administrador do servidor de licenças DRM deve verificar se o servidor de licenças do cliente conseguiu recuperar os arquivos CRL do Adobe. Se o cliente estiver usando o Tomcat, ele poderá verificar o diretório tomcat/temp/ para ver se há 4 arquivos .CRL.<ul><li>Se os arquivos estiverem nesse diretório, clique duas vezes nos arquivos no Windows Explorer e no aplicativo visualizador de CRL para determinar se algum dos arquivos expirou.</li><li>Se não houver arquivos em tomcat/temp/, pode-se supor que este servidor de licença nunca foi capaz de alcançar o servidor CRL Adobe devido a um problema de firewall ou roteamento.</li></ul>Para obter mais informações, consulte Regras de firewall.<br>Se os arquivos da CRL não estiverem disponíveis ou tiverem expirado, você deverá confirmar se o servidor de licenças pode ser acessado. Quando o problema de rede foi resolvido, os problemas do 3324 também foram resolvidos.<br>Se todos os 4 arquivos CRL existirem no diretório omcat/temp/ e os clientes ainda estiverem recebendo erros 3324, pode haver problemas de acesso a arquivos CRL.<br>Para resolver esse problema, talvez você queira revisar os logs e limpar os arquivos CRL existentes.<br>Se não houver problemas no servidor, solicite que o usuário redefina como descrito no Código de erro 3322. |
